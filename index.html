Explore, Explain, Propose

Introduction

Problem

Many interfaces treat behavior as static states. I’m interested in interfaces that respond like living systems: sensing change, stabilizing through feedback, and expressing rhythm.

Proposal

Translate behavioral primitives into a spatial interaction system for everyday contexts—using banking/financial signals as a concrete frame for urgency, stability, and change.

Framework

A biocybernetic lens: feedback, coupling, and self-organization. The goal is not to “animate data,” but to make data behave in interpretable, functional ways.

Methods

Start from observable biological behaviors, then formalize them into computational rules (attraction, repulsion, alignment, local awareness, relevance) that can be tuned and tested.

Prototype

A parameterized simulation that expresses distinct interaction “states,” plus a sensing pipeline that can translate environmental signals into real-time modulation.

Precedents

This work sits between data-as-material, responsive environments, and cybernetic systems thinking—where sensing and feedback become the interface.

  • Refik Anadol — Machine Hallucinations
  • Neri Oxman — Material Ecology
  • Ryoji Ikeda — data-verse
  • MIT Responsive Environments
  • Uexküll / Bateson / von Foerster — cybernetic feedback + Umwelt

Audience

For interaction designers, HCI researchers, and spatial computing builders exploring ambient, legible feedback loops. Also relevant to financial UX teams interested in conveying risk, stability, and attention without dashboards.

Trajectory

Next steps move from simulation toward interaction studies: analog experiments, refinement of the behavior engine, and a mixed-reality spatial interface prototype.
